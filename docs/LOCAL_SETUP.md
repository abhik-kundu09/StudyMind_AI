# StudyMind AI ‚Äî Local Development Guide

This guide covers everything needed to run StudyMind AI fully on your local machine.

---

## Prerequisites

| Tool | Version | Purpose |
|---|---|---|
| Docker Desktop | Latest | Runs MongoDB + Redis locally |
| Python | 3.14+ | Backend runtime |
| Node.js | 20+ | Frontend build |
| Groq API key | ‚Äî | Free LLM inference ‚Äî get at [console.groq.com](https://console.groq.com) |

---

## Step 1 ‚Äî Clone the repository

```bash
git clone https://github.com/abhik-kundu09/StudyMind_AI.git
cd StudyMind_AI
```

---

## Step 2 ‚Äî Start MongoDB + Redis

```bash
docker compose up -d
```

This starts:
- **MongoDB** on `localhost:27017`
- **Redis** on `localhost:6379`

Verify both are running:
```bash
docker compose ps
```

---

## Step 3 ‚Äî Backend setup

### Create virtual environment

```bash
cd backend

# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python3 -m venv venv
source venv/bin/activate
```

### Install dependencies

```bash
pip install -r requirements.txt
```

> First install takes 5‚Äì10 minutes ‚Äî sentence-transformers downloads the `all-MiniLM-L6-v2` embedding model (~90MB).

### Create `.env`

```bash
cp .env.example .env
```

Edit `backend/.env`:

```env
MONGODB_URI=mongodb://localhost:27017
MONGODB_DB_NAME=studymind
REDIS_URL=redis://localhost:6379
JWT_SECRET_KEY=<generate with: python -c "import secrets; print(secrets.token_hex(32))">
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=15
REFRESH_TOKEN_EXPIRE_DAYS=7
GROQ_API_KEY=<your key from console.groq.com>
FRONTEND_ORIGIN=http://localhost:5173
COOKIE_SECURE=false
ENVIRONMENT=development
DEBUG=true
```

### Start the API server

```bash
uvicorn app.main:app --reload --port 8000
```

API runs at `http://localhost:8000`. Swagger UI at `http://localhost:8000/docs`.

### Start the Celery worker (separate terminal)

Celery is required for PDF upload processing. Without it, uploaded PDFs will stay in `pending` status.

```bash
# Activate venv first, then:

# Windows (--pool=solo required on Windows)
python -m celery -A app.tasks.celery_app.celery_app worker --loglevel=info --pool=solo

# macOS / Linux
celery -A app.tasks.celery_app.celery_app worker --loglevel=info
```

## Step 4 ‚Äî Frontend setup

```bash
cd frontend
npm install
cp .env.example .env
```

`frontend/.env` should contain:
```env
VITE_API_URL=http://localhost:8000
```

Start the dev server:
```bash
npm run dev
```

App runs at `http://localhost:5173`.

## Common Issues

| Issue | Cause | Fix |
|---|---|---|
| PDF stays `pending` forever | Celery worker not running | Start Celery worker in a second terminal |
| `MONGODB_URI` connection error | Docker not running | Run `docker compose up -d` |
| `sentence-transformers` slow first request | Model downloading | Wait ~30s on first PDF upload ‚Äî cached after |
| Port 8000 already in use | Another process | Use `uvicorn app.main:app --reload --port 8001` and update `VITE_API_URL` |
| `ModuleNotFoundError` | venv not activated | Run `venv\Scripts\activate` (Windows) or `source venv/bin/activate` |
| CORS error in browser | `FRONTEND_ORIGIN` mismatch | Confirm `.env` has `FRONTEND_ORIGIN=http://localhost:5173` |

## Stopping services

```bash
# Stop Docker containers
docker compose down

# Stop API server and Celery worker
# Press Ctrl+C in each terminal
```

## Project data locations

| Data | Location | Notes |
|---|---|---|
| MongoDB data | Docker volume `mongo_data` | Persists across restarts |
| Redis data | In-memory | Cleared on container restart |
| FAISS indexes | `backend/data/faiss_indexes/` | Gitignored ‚Äî regenerated by re-uploading PDFs |
| HuggingFace model cache | `~/.cache/huggingface/` | Downloaded once, reused |
